Current Problem
---------------
The frontend only receives the finished video after processing completes.

I want to upgrade the application into a real-time traffic monitoring system similar to an NCAP operator dashboard.

Goal
----
While the backend is processing the uploaded video, the frontend should receive annotated frames continuously and display them as a live monitoring feed.

The final MP4 should still be generated after processing finishes.

Backend Requirements
--------------------

Keep the current background job architecture.

Create a FastAPI WebSocket endpoint:

/ws/tracking/{job_id}

Inside the tracking loop:

For every processed frame:

- Run YOLOv8 tracking.
- Draw:
    - Bounding boxes
    - Vehicle class
    - Tracking ID
    - Confidence

Encode the annotated frame using:

cv2.imencode(".jpg", frame)

Convert to Base64.

Send through the WebSocket.

Example message:

{
    "frame": "<base64 jpeg>",
    "frame_number": 245,
    "processing_fps": 21.5,
    "active_tracks": [...],
    "vehicle_statistics": {...},
    "progress": 48
}

Continue writing every frame to the output MP4.

Do not remove the existing progress tracking.

Keep the existing polling endpoint because React will still use it for job completion.

Frontend Requirements
---------------------

Create a reusable WebSocket hook.

Connect to:

ws://localhost:8000/ws/tracking/{job_id}

Whenever a new frame arrives:

Decode the Base64 JPEG.

Display it inside a responsive video container.

Update:

- Current frame
- Processing FPS
- Progress
- Active tracking IDs
- Vehicle statistics

When processing completes:

- Close the WebSocket.
- Replace the live feed with the final processed MP4.
- Continue displaying statistics.
