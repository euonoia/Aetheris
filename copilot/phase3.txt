I am building a YOLOv8 learning project.

Current stack:
- Backend: FastAPI
- Frontend: React + Vite + TypeScript
- Object Detection: Ultralytics YOLOv8

Current Progress:
✅ Image upload
✅ Image detection
✅ Annotated image display
✅ Detection metadata returned
✅ Vehicle filtering
✅ Confidence threshold
✅ Vehicle statistics

Now I want to implement Phase 3: Video Detection.

Goal:
Allow users to upload a video, process it frame-by-frame using OpenCV and YOLOv8, generate an annotated output video, and display it in React.

Backend Requirements:
- Create a new POST /detect-video endpoint.
- Accept uploaded video files.
- Save videos inside app/uploads.
- Read the uploaded video using cv2.VideoCapture.
- Create an output video using cv2.VideoWriter.
- Process the video one frame at a time.
- Run YOLOv8 inference on every frame.
- Only keep vehicle detections:
  - Car (2)
  - Motorcycle (3)
  - Bus (5)
  - Truck (7)
- Draw bounding boxes and labels on each processed frame.
- Write each processed frame into the output video.
- Save the annotated video inside app/outputs.
- Return a JSON response including:
  - filename
  - video_url
  - processing_time
  - total_frames
  - total_vehicle_detections
  - vehicle_statistics
- Expose app/outputs using FastAPI StaticFiles if not already configured.

Frontend Requirements:
- Continue using React + Vite + TypeScript.
- Allow users to upload either an image or a video.
- Automatically detect whether the uploaded file is an image or a video.
- If it is an image:
    - Continue using the existing image detection workflow.
- If it is a video:
    - Call /detect-video.
    - Display the processed video using an HTML <video controls> element.
    - Display:
        - processing time
        - total frames processed
        - total vehicle detections
        - vehicle statistics
- Keep API logic inside services/api.ts.
- Keep TypeScript interfaces organized.
- Handle loading and error states.

Constraints:
- Do not implement tracking.
- Do not implement ByteTrack.
- Do not implement vehicle counting.
- Do not implement databases.
- Keep the code modular.
- Add comments explaining every important OpenCV and YOLO step because this project is for learning.
